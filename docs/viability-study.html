1. Introducción y Objetivos

Nexus es una aplicación móvil y web que combina lo mejor de las plataformas de ofertas y de segunda mano, ofreciendo un espacio donde los usuarios pueden descubrir ofertas destacadas, productos usados y promociones de empresas en un solo lugar.

El objetivo principal es crear un espacio interactivo y visualmente atractivo para:

  • Buscar productos nuevos y usados.
  • Publicar anuncios propios.
  • Interactuar socialmente (likes) para destacar ofertas.
  • Utilizar un sistema de envíos integrado (Nexus Go).

2. Alcance Preliminar

El proyecto busca desarrollar una aplicación funcional para Android (y posteriormente iOS) y Web con:

  • Inicio con ofertas destacadas y categorías.
  • Apartado de Segunda Mano.
  • Sección de Destacados (basada en likes).
  • Buscador global (nuevos y usados).
  • Perfiles de usuario y gestión de publicaciones.
  • Módulo Nexus Go para envíos con comisión.

Exclusiones iniciales: No se incluyen pasarelas de pago externas ni funcionalidades avanzadas de chat en la v1.

3. Viabilidad Técnica

  • Tecnologías: Frontend (Angular, Android Studio - Java), Backend (Java Spring Boot, NodeJS, PostgreSQL).
  • Compatibilidad: Dispositivos Android, Web (futuro iOS).
  • Recursos humanos: Se requiere un equipo de 3 desarrolladores con experiencia en las tecnologías mencionadas.

4. Viabilidad Económica

  • Costes estimados iniciales: Desarrollo (6k-8k €), Servidores (~500 €/año), Marketing inicial (1k €).
  • Fuentes de financiación: Fondos propios, microinversores.
  • Monetización futura: Comisiones por Nexus Go, publicidad destacada para empresas.
  • Análisis costo-beneficio: Inversión inicial moderada con potencial de retorno a medio plazo si se consigue tracción de usuarios.

5. Viabilidad Operativa

  • Impacto en usuarios: Experiencia simple, moderna, con incentivos sociales.
  • Impacto en empresas: Canal para promocionar productos.
  • Procesos internos: Se necesitará un panel de administración para gestión de contenido y usuarios.

6. Viabilidad Legal y Normativa

  • Cumplimiento estricto del RGPD.
  • Términos y condiciones claros.
  • Posible verificación de identidad para vendedores de segunda mano.
  • Protección de propiedad intelectual.

7. Riesgos Iniciales

Riesgo Descripción Medida de mitigación
Competencia alta Existen apps similares consolidadas Diferenciación por integración de funciones y sistema de likes
Falta de usuarios iniciales Baja adopción al principio (problema huevo/gallina) Campañas de lanzamiento, bonificaciones, enfoque local inicial
Publicaciones fraudulentas Riesgo de estafas en segunda mano Moderación (IA y manual), sistema de reportes y valoraciones
Costes operativos Mantenimiento de infraestructura Escalado progresivo de servidores según la base de usuarios

8. Conclusión y Recomendación

El proyecto Nexus presenta una propuesta de valor interesante al unificar dos mercados complementarios. Es viable desde el punto de vista técnico y económico, siempre que se gestione eficientemente el desarrollo y se implemente una estrategia de captación de usuarios inicial efectiva.

Se recomienda: Continuar con el desarrollo de un prototipo funcional (MVP) y realizar una prueba piloto en un mercado local limitado para validar la aceptación y ajustar la estrategia antes de un lanzamiento a mayor escala.
